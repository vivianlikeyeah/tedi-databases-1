Building Databases

To start off with, we’re going to be using pure intuition and microsoft excel to build our databases. In the coming nodes, we will dive into appropriate database management systems, as well as techniques to design, normalise and build our databases.

Activity

In our previous example we created one table for all of the students at TEDI London.Now we’re going to create another table that we can LINK to this table. This table is going to be called Nationalities (you can open a new tab in excel) In this table create 5 fields.
Nationality, Country, Hemisphere, Time Zone, Language
Then populate it with all of the data for each country that was listed in your first table. If a country was repeated twice, you only need to list it once. Your Primary Key is Nationality. Your foreign key in the first table is also Nationality. Now we have two tables, that are linked by the field Nationality. For example, if i’m looking at the student database, and I want to know ‘what time is it where sam is?’ I can cross check her nationality - Australian, with the nationalities database, and check what the time zone is where all my australian friends are.

Activity

Design a database (decide how many tables, and the fields in each table), based off the following customer spec. Jamil is throwing a huge birthday spectacular to celebrate their birthday. They need to keep track of all the guests coming along. Jamil needs to know their first and last name so that the security can tick off the appropriate person on the list. They also need to make sure that all of their guests are of age, so this needs to be recorded. They need to keep track of each guests dietary requirements. They have planned a full menu, so they need to know each dish,how many servings of each dish they have, and whether each dish is Vegetarian, Vegan, Nut-Free and/or Dairy Free. Compare your tables with someone else at TEDI London. Do you have the same tables? What different design decisions did you make and why?
